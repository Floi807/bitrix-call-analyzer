# Bitrix24 Call Transcript to Deal Products

Локальное PHP-приложение для Bitrix24, которое:

- получает расшифровку звонка, связанную со сделкой
- распознает упомянутые товары, количество и параметры
- автоматически обновляет товарные строки сделки
- дает ручную кнопку запуска в карточке сделки

Приложение не требует Composer и сторонних SDK: вся REST/OAuth-логика реализована на чистом PHP.

## Что важно про Bitrix AI API

По официальной документации Bitrix24 на 10 апреля 2026 года раздел REST `ai` предназначен для регистрации собственного AI-движка (`ai.engine.register`, `ai.engine.list`, `ai.engine.unregister`), а не для прямого вызова встроенного Bitrix CoPilot как публичного completion/chat API из REST-приложения.

Поэтому в проекте:

- рабочий режим по умолчанию реализован без внешних AI-сервисов
- анализатор построен на rule-based разборе текста, словаре синонимов и каталожном матчинге
- предусмотрен слой `AiExtractorInterface`, чтобы позже подключить Bitrix AI-интеграцию, если в вашем портале есть доступный внутренний endpoint или собственный AI engine

## Возможности

- автоматический запуск по событию обновления сделки
- ручной запуск из вкладки в карточке сделки
- поиск расшифровки из нескольких источников
- защита от повторной обработки одного и того же текста
- логирование обработок

## Предпосылки

- PHP 8.1+
- HTTPS-домен, доступный из Bitrix24
- права приложения на `crm`, `catalog`, `placement`

## Быстрый старт

1. Скопируйте `config/app.php.dist` в `config/app.local.php`.
2. Заполните `app_url`, `client_id`, `client_secret`.
3. При необходимости настройте `transcript_sources` и `catalog.aliases`.
4. Разместите приложение на PHP-хостинге.
5. В Bitrix24 создайте локальное или публичное приложение с основным URL `https://your-domain.example/public/index.php`.
6. Установите приложение в портал.
7. Откройте сделку и используйте вкладку `Call Analyzer`.

## Откуда берется расшифровка

По умолчанию приложение пытается получить текст в таком порядке:

1. из последней связанной CRM-активности типа звонок
2. из поля `COMMENTS` сделки
3. из пользовательских полей, перечисленных в `transcript_sources.deal_fields`

Если в вашем портале расшифровка попадает в отдельное поле сделки, просто добавьте его в конфиг:

```php
'transcript_sources' => [
    'deal_fields' => ['UF_CRM_CALL_TRANSCRIPT', 'UF_CRM_1234567890'],
    'activity_types' => [2],
],
```

## Ограничения

- В текущей среде `php` не установлен, поэтому автоматическую проверку синтаксиса я здесь не выполнил.
- В публичной REST-документации Bitrix24 нет стабильного documented endpoint для прямого prompt/completion вызова встроенного Bitrix AI. Поэтому рабочий режим сделан без обязательного AI-вызова.
- Формат хранения расшифровки зависит от конкретного портала и телефонии. Если текст не хранится в CRM-активности, настройте пользовательское поле сделки или внешний POST в `?action=manual-sync`.
